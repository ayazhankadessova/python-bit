## 2. Format Specifiers: The Building Blocks

Format specifiers are placeholders that tell Python what type of data to expect:

| Specifier | Description | Example |
|-----------|-------------|---------|
| `%s` | String | `"Hello, %s" % "World"` |
| `%d` | Integer (decimal) | `"Level: %d" % 5` |
| `%f` | Float (decimal number) | `"Score: %f" % 95.5` |
| `%.2f` | Float with precision | `"Accuracy: %.2f%%" % 98.76` |

Let's experiment with different format specifiers:

<PythonCodeEditor 
  initialCode={`# Experiment with different format specifiers

level = 42
score = 98.7654

# String formatting
print("Player: %s" % player_name)

# Integer formatting
print("Level: %d" % level)

# Float formatting (default)
print("Raw score: %f" % score)

# Float with 2 decimal places
print("Formatted score: %.2f" % score)

# Float as percentage with 1 decimal place
print("Completion: %.1f%%" % score)

# Try changing the values or formatting options!`}

## 3. Multiple Value Formatting

When formatting multiple values, place them in parentheses in the same order as the format specifiers:

## 4. F-Strings: Modern Python Formatting

Since Python 3.6, we have f-strings - a more readable way to format strings:

<PythonCodeEditor 
  initialCode={`# Modern f-string formatting

# Basic f-string
print(f"Player {name} (Level {level})")

# F-strings with expressions
print(f"HP: {health}/{max_health} ({health/max_health:.0%})")

# F-strings with formatting options
money = 1234.56
print(f"Gold: {money:.2f}")

## 7. Common Pitfalls and Debugging

When working with string formatting, watch out for these common issues:

<PythonCodeEditor 
  initialCode={`# Common string formatting pitfalls

# 1. Mismatched format specifiers
player = "Alex"
level = 42
# Uncomment to see the error:
# print("Player %d is at level %s" % (player, level))

# 2. Missing parentheses for multiple values
# Uncomment to see the error:
# print("Stats: %s %d" % player, level)

# 3. Wrong number of values
# Uncomment to see the error:
# print("Player: %s, Level: %d, Health: %d" % (player, level))

# 4. Using % without escaping
health_percent = 75
# This will cause an error because % is a special character
# Uncomment to see the error:
# print("Health: %d%" % health_percent)

# Fixed version:
print("Health: %d%%" % health_percent)

# 5. Using format specifiers with f-strings (mixing styles)
# Uncomment to see the error:
# print(f"Player: %s" % player)

# Fixed version:
print(f"Player: {player}")

# Try fixing the commented lines!`}
